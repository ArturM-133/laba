# Отчёт о качестве данных

## 1. Введение

**Датасет:** Розничные продажи  
**Источник:** https://www.kaggle.com/datasets/mohammadtalib786/retail-sales-dataset  
**Размер:** 1000 × 9  

**Предметная область:** Набор данных содержит информацию о транзакциях, клиентах и товарах, включая демографические признаки и финансовые показатели. Эти данные позволяют анализировать покупательское поведение и финансовые итоги каждой сделки.  

**Условный заказчик:** Коммерческий директор, отдел маркетинга и команда по закупкам розничной сети для анализа поведения покупателей, оптимизации ассортимента и повышения эффективности продаж.  

**Возможные задачи ИАД:**
1. Прогнозирование спроса
2. Сегментация клиентов по покупательскому поведению
3. Анализ ассоциативных правил для оптимизации ассортимента и кросс-продаж

## 2. Паспорт датасета

| Столбец | Тип | Описание                            | Примечания                    |
|---------|-----|-------------------------------------|-------------------------------|
| Transaction ID | int64 | Идентификатор для каждой транзакции | Нет пропусков                 |
| Date  | str | Дата совершения транзакции          | Нет пропусков                 |
| Customer ID  | str | Идентификатор для каждого клиента   | Нет пропусков                 |
| Gender  | str | Пол покупателя                          | Категориальный, нет пропусков |
| Age  | int64 | Возраст клиента                          | Нет пропусков                 |
| Product Category  | str | Категория приобретаемого товара                          | Категориальный, нет пропусков |
| Quantity  | int64 | Количество приобретенных единиц товара                          | Нет пропусков                    |
| Price per Unit  | int64 | Цена одной единицы продукта                         | Нет пропусков                    |
| Total Amount  | int64 | Общая денежная стоимость сделки                          | Нет пропусков                   |

## 3. Основные проблемы качества данных

| № | Тип проблемы          | Описание                                                                          | Потенциальный риск                                    | Предлагаемое решение                        |
|---|-----------------------|-----------------------------------------------------------------------------------|-------------------------------------------------------|---------------------------------------------|
| 1 | Пропуски              | 0 пропусков                                                                       | Отсутствует                                           | Не требуется                                |
| 2 | Дубликаты             | 0 полных дубликатов, 0 дубликатов по ключевым полям (Transaction ID, Customer ID) | Отсутствует                                           | Не требуется                                |
| 3 | Выбросы               | 0 выбросов                                                                        | Отсутствует                                           | Не требуется                                |
| 4 | Формат даты           | Дата хранится в строковом формате                                                 | Невозможность временного анализа, извлечения периодов | Преобразовать в datetime: pd.to_datetime()  |
| 5 | Категориальные данные | Gender и Product Category требуют кодирования                                     | Проблемы с алгоритмами ML, требующими числовых данных | One-Hot Encoding или Label Encoding         |
